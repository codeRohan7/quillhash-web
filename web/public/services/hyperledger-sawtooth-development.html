Features of Hyperledger Sawtooth

  • Ethereum Smart Contract Compatibility
  • Parallel Transactions
  • Validator
  • Private Network
  • Separation System
  • Event System
Sawtooth has a kind of transaction processor — known as a Seth transaction processor which is based on the Hyperledger Burrow Ethereum Virtual Machine (EVM). This enables it to run Ethereum smart contract written in a language known as Solidity.
A unique feature of Sawtooth is its ability to group transactions into batches. A batch of transactions can be packed into a block. Thus, affording Sawtooth the ability to handle complex transactions. There is Atomicity, Consistency, Isolation and Durability (ACID) principle in Sawtooth.
Just like a miner, there’s a validator in sawtooth. The job of the validator is to check and ensure that a transaction (or a message indicating the transaction to be executed) has a valid signature and then direct it to the appropriate transaction processor.
Hyperledger Sawtooth overcome the challenges of permissioned (private) network. Clusters of Sawtooth nodes can be easily deployed with separate permissioning. There is no centralized service that could potentially leak transaction patterns or other confidential information.
Sawtooth makes it easy to develop and deploy an application by providing a clear separation between the application level and the core system level. Sawtooth provides smart contract abstraction that allows application developers to write contract logic in a language of their choice.
Hyperledger Sawtooth enable a feature where application can create and broadcast events. This allows application to subscribe to certain event related to blockchain or can subscribe to specific application event.
